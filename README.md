# 📊 End-to-End Sales Forecasting & Demand Intelligence System

A complete, production-style data science project built on the Superstore Sales dataset:
time-series forecasting, anomaly detection, product demand segmentation, and an interactive
dashboard — the kind of system a retail data science team actually builds and maintains.

---

## 🎯 Problem Statement

Retailers live or die by one question: *how much of each product will we sell next month, and
will we have enough stock to meet that demand?* This project builds an intelligent forecasting
system that predicts future demand, flags unusual sales patterns, segments products by demand
behavior, and presents everything through a dashboard a business manager could act on.

---
## ⭐ Project Highlights

- End-to-end retail sales forecasting pipeline
- Time-series forecasting using SARIMA, Prophet, and XGBoost
- Product demand segmentation using K-Means Clustering
- Sales anomaly detection using Isolation Forest and Z-Score
- Interactive Streamlit dashboard
- Executive business report for non-technical stakeholders

## 🧠 What's Inside `analysis.ipynb`

1. **Data Loading, Merging & Deep Exploration** — cleaning, time-feature engineering, and 4 data-backed business questions
2. **Time Series Analysis & Decomposition** — trend/seasonal/residual breakdown + stationarity (ADF) testing
3. **Sales Forecasting — 3 Models** — SARIMA, Facebook Prophet, and XGBoost, compared on MAE/RMSE/MAPE with a data-driven recommendation
4. **Category & Region-Level Forecasting** — the winning model re-applied to 5 business segments
5. **Anomaly Detection** — Isolation Forest and rolling Z-Score methods, cross-compared
6. **Product Demand Segmentation** — K-Means clustering (with elbow method + PCA visualization) into actionable stocking-strategy groups
7. **Deployment** — the Streamlit dashboard in `app.py`
8. **Executive Business Report** — [summary.docx](summary.docx)

## 🚀 Running This Project

### 1. Requirements
- Python 3.10 or later

### 2. Set up the environment
This project's `.venv` is not included in the repo — create your own:

**Windows (PowerShell):**
```bash
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

**macOS / Linux:**
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3. Explore the analysis
Open `analysis.ipynb` in Jupyter Notebook, JupyterLab, or Google Colab. All cells are already
executed — outputs and charts are visible without re-running anything — but the notebook can
be re-run end-to-end from a clean kernel.

### 4. Run the dashboard locally
```bash
streamlit run app.py
```
This starts the app at `http://localhost:8501/` (see [Demo](#️-demo) above).

### 5. Deploy to Streamlit Community Cloud (free)
Push this folder to a GitHub repository, then connect it at
[share.streamlit.io](https://share.streamlit.io) and point it at `app.py`. The app computes
everything directly from `train.csv` (with caching), so it needs no other setup.

## ⚠️ Known Limitation

All forecasting models are trained on 4 years of monthly history (48 data points). This is
sufficient to capture a reliable yearly seasonal pattern but is a thin sample for a statistical
model, especially since a handful of unusually large one-off orders can shift a given month's
total noticeably (see the anomaly detection section). Forecasts should be treated as a rolling,
quarterly-refreshed guide and re-validated as new months of actuals arrive.
